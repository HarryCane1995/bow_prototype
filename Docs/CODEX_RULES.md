# Codex Rules

Правила для будущих изменений в проекте:

- Не делать монолитный `PlayerController`.
- Не менять несвязанные системы без необходимости.
- Не ломать существующие exported-поля.
- Все новые игровые настройки выносить в Inspector через `[Export]`.
- Перед изменениями понимать текущую структуру сцены.
- После изменений писать список изменённых файлов.
- При создании новых систем создавать отдельные модули.
- Не удалять рабочие сцены и ассеты.
- Не переименовывать публичные классы/ноды без необходимости.
- Маленькие итерации предпочтительнее больших переписываний.

## Практический чеклист

Перед правками:

- Проверить relevant `.tscn` и `.cs` файлы.
- Понять, какие ноды и exported-поля уже используются в Inspector.
- Ограничить изменение задачей пользователя.

После правок:

- Перечислить изменённые файлы.
- Указать, менялись ли сцены, ассеты или только код/документация.
- Если запуск или тест невозможен, явно написать почему.
